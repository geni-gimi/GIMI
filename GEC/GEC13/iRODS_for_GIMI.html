Tutorial on how to setup iRODS for use within GIMI

This page describes the installation steps required to setup and Integrated Rule-Oriented Data System 
(iRODS) server for the GIMI project. 

General information on iRODS can be found here.

Here are the steps necessary to perform the installation:
  1. Install iRODS as described under "Quick Start iRODS Install Instructions"at the following link:
    https://www.irods.org/index.php/Installation



  2. Configure iRODS server by running "irodssetup" in the root directory of your iRODS installation.
    Note iCAT does not have to be installed since RENCI runs iCAT on its iRODS server.
    The hostname for the RENCI iRODS server is: geni-gimi.renci.org
    Make sure to contact Shu Huang (shuang@renci.org) to make sure RENCI's firewall is
    configured to allow your server to connect to geni-gimi.renci.org

    Here's the server section from config/irods.config on the UMass server:
    $IRODS_HOME = '/nfs/emmy8/scratch3/iRODS';
    $IRODS_PORT = '1247';
    $SVR_PORT_RANGE_START = '20000';
    $SVR_PORT_RANGE_END = '20199';
    $IRODS_ADMIN_NAME = 'rods';
    $IRODS_ADMIN_PASSWORD = 'rods';
    $IRODS_ICAT_HOST = 'geni-gimi.renci.org';

    $DB_NAME = 'ICAT';
    $RESOURCE_NAME = 'iRODSUmass';
    $RESOURCE_DIR = '/nfs/emmy8/scratch3/iRODS/Vault';
    $ZONE_NAME = 'geniRenci';
    $DB_KEY = '123';

    $GSI_AUTH = '0';
    $GLOBUS_LOCATION = '';
    $GSI_INSTALL_TYPE = '';

  3. Start your local  iRODS server with "irodsctl start".
    ("irodsctrl stop" will shut it down.)

  4. iadmin is the command line administration tool for iRODS
Getting files in and out of iRODS:

  1. iRODS client commandsa can be found at:
    iRODS/clients/icommands/bin

  2. ihelp lists and briefly explains all available client commands. Help for this command can also be
    found online here.


  3. Example: A simple way to add a file into iRODS is iput.
